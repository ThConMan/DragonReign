Documentation

Everything DragonReign does, in plain English. Built for Paper / Minecraft 26.1.2, Java 21+.

Overview

DragonReign makes the Dragon Egg a single, contested trophy. There's only ever one egg. It can't be hidden in containers, it can't be sat on forever, and holding it earns rewards and a permanent Dragonlord title. Every rule is on by default and can be flipped in-game with /dr gui.

All protections apply to every dragon egg item and block — there's no way to launder the rules by crafting a look-alike.

Install

  1. Download DragonReign-x.y.z.jar from the latest release.
  2. Drop it in your server's plugins/ folder (remove any older DragonReign jar).
  3. Restart the server.
  4. Adjust plugins/DragonReign/config.yml or run /dr gui. That's it.

Optional soft-depends (all auto-detected): PlaceholderAPI, TAB, LuckPerms, CMI, EssentialsX, Vault.

The rules

No containers

The egg can only sit in a player inventory or be placed as a block — never a chest, ender chest, bundle, hopper, minecart, or any container.

Drop control

Dropping is a toggle (no-drop). When drops are allowed, a loose egg is fireproof (immune to lava, fire, cactus, explosions), floats out of lava and water, and never despawns — but still dies in the void and respawns at the End.

Respawn on inactivity

If the keeper is gone for inactivity-days (default 14), the egg respawns at the End portal.

Staleness respawn

If the egg sits untouched for staleness-days (default 10) it respawns even while the owner is online — no parking it forever.

Respawn countdown

The respawn isn't instant: a timed countdown warns the server so players can race to the End. If the keeper returns in time, it's cancelled.

Egg compass

Players near a placed egg get an action-bar arrow pointing at it — warmer the closer they get.

Hold rewards

The keeper earns escalating, configurable rewards for actively holding the egg. The ladder resets when the egg changes hands.

Anti-AFK

Rewards and Dragonlord time only build while you're truly playing. AFK pools and auto-clickers don't count.

Never lost

A loose egg never despawns and is fireproof; if it falls into the void it's rebuilt at the End. There is always exactly one egg.

Dragonlord aura

Hold the egg long enough (default 7 days of active time) to permanently earn the Dragonlord title and a living aura: shimmering spirit-orbs that wander around you, livewire surges, lightning arcs between orbs, a coronation flourish, and reactivity to combat — every layer configurable.

Staff inbox

Alt/IP flags, dupe cleanups, and respawns alert online admins and queue for the rest.

History log

Every meaningful egg event is recorded and browsable in-game.

Config

The whole config.yml is commented in plain language, and most of it is editable live from /dr gui. The key knobs:

respawn-on-inactivity:
  inactivity-days: 14      # keeper gone this long -> egg respawns
  staleness-days: 10       # egg untouched this long -> respawns anyway

rewards:
  interval-minutes: 60     # active holding time per reward tier
  tiers:                   # each tier = console commands ({player}, {tier})
    - ["give %player% diamond 4"]
    - ["give %player% diamond 8", "xp add %player% 30 levels"]

victor:
  threshold-hours: 168            # active hold time to become a Dragonlord (7 days)
  title: "&6Dragonlord"           # the title (a rendered gradient by default)
  particle: DUST_COLOR_TRANSITION # aura motes; shimmer aura-color -> aura-color-2
  aura-color: '#9D4EDD'           # gradient start   (aura-color-2: '#4ED9FF' = end)
  orbit-count: 7                  # wandering spirit-orbs around the player
  livewire-intensity: 1.0         # how hard the orbs occasionally surge (0 = calm)
  combat-reactivity: true         # aura intensifies when the holder fights/sprints/is hurt
  arcs-enabled: true              # lightning arcs between nearby orbs
  coronation-enabled: true        # one-time flourish on becoming a Dragonlord

void-safety:
  enabled: true                   # loose egg never despawns; void drops rebuild at the End
  fireproof: true                 # loose egg immune to lava/fire; floats out of lava & water

chat:
  enabled: false                  # turn on if your chat plugin doesn't show the title

In-game menus

  • Settings (/dr gui) — flip every rule on/off, set the inactivity/staleness days, cycle the announcement sound, jump to the placed egg, and open the other menus.
  • History (/dr log) — newest-first list of every egg event, with who, where, and when.
  • Inbox (/dr inbox) — staff alerts; click to mark read, shift-click to dismiss.

Showing the Dragonlord title

The title is published through PlaceholderAPI as %dragonreign_title%, so any plugin that reads PlaceholderAPI can display it. Put that placeholder right after the rank prefix, before the name.

  • TAB — add %dragonreign_title% to each group's tabprefix (tab list) and tagprefix (above-head nameplate) in TAB/groups.yml.
  • Most chat / nameplate plugins — drop %dragonreign_title% into their name format.
Some chat handlers (notably CMI's display name) don't expand external PlaceholderAPI placeholders. For those, set chat.enabled: true and DragonReign adds the title to a victor's chat line itself.

Placeholders

With PlaceholderAPI installed, DragonReign exposes these — drop them into TAB, a scoreboard sidebar, chat, or a nameplate:

PlaceholderShows
%dragonreign_title%The Dragonlord title (empty for non-victors).
%dragonreign_owner%The egg holder's name, or none.
%dragonreign_reset_in%Friendly time until the egg next resets, e.g. 5d 3h (or —).
%dragonreign_reset_seconds%Raw seconds until reset (for scoreboard math), or -1.
%dragonreign_inactivity_in% · %dragonreign_staleness_in%The two reset timers, separately.
%dragonreign_countdown%Seconds left in an active respawn countdown (0 if none).
%dragonreign_reward_tier%The owner's current reward tier.
%dragonreign_is_victor% · %dragonreign_has_owner% · %dragonreign_countdown_active%Booleans (true/false).
Example TAB sidebar lines:
'&5✦ &fDragon Egg: %dragonreign_owner%'
'&5✦ &fResets in: %dragonreign_reset_in%'

FAQ

Can the egg ever be lost or duped?

No. There's always exactly one egg; void drops are rebuilt at the End, and the dupe/exploit paths are closed.

Does it lag the server?

No. Everything keys off a single egg with interval-based checks — no per-tick scans.

Do I need PlaceholderAPI / TAB / CMI?

No. They're optional hooks. DragonReign runs fine on a bare server.

How do I test it as an admin?

Ops bypass protections, so test from a non-op alt or use /dr victor grant and /dr respawn force.
